# 智慧災害通報系統 - 功能需求

## 系統目標
讓民眾在地震、颱風、水災等災情發生時，透過 LLM 對話介面快速通報災情。系統自動蒐集整理災情詳細資訊，存入資料庫，並呈現在地圖中。

---

## 核心功能

### F1: LLM 對話通報
- 民眾進入通報頁面，與 AI 助手對話
- Claude 以繁體中文、同理冷靜的語氣引導民眾描述災情
- 逐步收集：災情種類、地點、嚴重程度、傷亡人數、發生時間等
- 地點引導到具體地址或地標，以便地理編碼
- 使用 Claude Tool Use 將對話擷取為結構化資料
- 資料擷取後顯示摘要供民眾確認
- 確認後提交通報

### F2: 結構化資料擷取
LLM 需從對話中擷取以下欄位：
- **災情種類**: earthquake/typhoon/flood/landslide/fire/other
- **災情描述**: 自由文字描述
- **地點文字**: 具體地址、路名、地標
- **經緯度**: 若 LLM 能推斷則填入，否則由 Geocoding 服務補充
- **嚴重程度**: 1(輕微) ~ 5(極嚴重)
- **傷亡人數**: 死亡人數
- **受傷人數**: 受傷人數
- **受困人數**: 受困人數
- **發生時間**: 災情發生的時間
- **通報者姓名**: 可選
- **通報者電話**: 可選

### F3: 事件去重 (Deduplication)
- 新通報進入時，自動判斷是否與近期已登錄的災情為同一事件
- 判斷維度：
  1. **空間**：PostGIS ST_DWithin 查詢半徑內事件
     - 地震: 50km
     - 颱風/水災: 20km
     - 火災/坍方: 5km
  2. **時間**：只比對最近 72 小時內的 active 事件
  3. **類型**：災情種類必須相同
  4. **LLM 判斷**：將新通報與候選事件交給 Claude 比對
- 同一事件 → 更新事件（取最新/最嚴重數據），通報關聯至該事件
- 不同事件 → 建立新的災情事件

### F4: 地圖總覽
- Leaflet 地圖顯示所有 active 災情事件
- 不同災情種類使用不同顏色/圖示 marker
- 點擊 marker 顯示事件摘要 popup
- 依災情種類、嚴重程度篩選
- Marker clustering（事件聚合）

### F5: 災情列表與搜尋
- 表格列出所有災情事件
- 關鍵字搜尋：標題、描述、地點
- 篩選：災情種類、嚴重程度、狀態、時間範圍
- 排序：時間、嚴重程度、通報數量
- 分頁瀏覽

### F6: 災情詳情與更新
- 查看事件完整資訊
- 查看所有關聯通報
- 手動更新災情狀態：active → monitoring → resolved
- 手動修正災情資訊
- 查看事件時間軸

---

## 非功能性需求
- 前端響應式設計（手機、桌面都能使用）
- Chat API 支援 streaming 回應（逐字顯示 AI 回覆）
- Geocoding 使用 Nominatim 免費 API（地址文字 → 座標）
- 備援：使用者可在地圖上手動點選定位
