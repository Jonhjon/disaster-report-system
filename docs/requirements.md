# 智慧災害通報系統 - 功能需求

## 系統目標
讓民眾在地震、颱風、水災等災情發生時，透過 LLM 對話介面快速通報災情。系統自動蒐集整理災情詳細資訊，存入資料庫，並呈現在地圖中。管理人員可透過管理中心端進行災情管理與監控。

---

## 系統架構

系統分為三個部分：
- **民眾端** (`frontend-public/`)：免登入，提供通報功能和災情地圖瀏覽
- **管理中心端** (`frontend-admin/`)：需登入，提供完整的災情管理和系統監控
- **後端 API** (`backend/`)：統一的 REST API，部分端點需 JWT 認證

---

## 民眾端功能

### F1: LLM 對話通報
- 民眾進入通報頁面，與 AI 助手對話
- Claude 以繁體中文、同理冷靜的語氣引導民眾描述災情
- 逐步收集：災情種類、地點、嚴重程度、傷亡人數、發生時間等
- 地點引導到具體地址或地標，以便地理編碼
- 使用 Claude Tool Use 將對話擷取為結構化資料
- 系統自動完成去重判斷，若偵測到相似事件則顯示候選卡片供民眾選擇合併或新建
- 通報完成後顯示摘要

### F2: 結構化資料擷取
LLM 需從對話中擷取以下欄位：
- **災情種類**: trapped/road_collapse/flooding/landslide/small_landslide/building_damage/utility_damage/fire/other
- **災情描述**: 自由文字描述
- **地點文字**: 具體地址、路名、地標
- **嚴重程度**: 1(輕微) ~ 5(極嚴重)
- **傷亡人數**: 死亡人數、受傷人數、受困人數
- **發生時間**: 災情發生的時間
- **通報者姓名/電話**: 可選

### F3: 唯讀災情地圖
- Leaflet 地圖顯示所有通報中（reported）的災情事件
- 不同災情種類使用不同顏色 marker
- 點擊 marker 顯示事件摘要（標題、類型、嚴重程度、通報數、時間）
- 依災情種類、嚴重程度、狀態篩選
- 民眾端地圖為唯讀，無編輯/刪除/修正地址功能

### F4: 使用說明
- 系統介紹、通報操作說明、地圖色碼對照

## 管理中心端功能

### F5: 登入認證
- 管理員帳號密碼登入（JWT Bearer Token）
- 登入後 token 存於 localStorage，所有 API 請求自動帶認證 header
- Token 過期或無效自動導向登入頁
- 支援登出功能

### F6: 災情地圖總覽（管理版）
- 與民眾端地圖相同的瀏覽功能
- 額外支援：位置估算事件的地址修正功能（PATCH /api/events/{id}/location）

### F7: 災情列表與搜尋
- 表格列出所有災情事件
- 關鍵字搜尋：標題、描述、地點
- 篩選：災情種類、嚴重程度、狀態、時間範圍
- 排序：時間、嚴重程度、通報數量
- 分頁瀏覽

### F8: 災情詳情與更新
- 查看事件完整資訊
- 查看所有關聯通報
- 手動更新災情狀態：reported → in_progress → resolved
- 手動修正災情資訊（標題、嚴重程度、傷亡人數、描述）
- 刪除事件（含確認機制）

### F9: LLM 呼叫日誌
- 顯示最近 100 筆 AI 模型呼叫紀錄
- 欄位：時間、模型名稱、延遲(ms)、輸入/輸出 tokens、狀態
- 可展開查看完整 prompt 和回應內容

## 後端功能

### F10: 事件去重 (Deduplication)
- 新通報進入時，自動判斷是否與近期已登錄的災情為同一事件
- 判斷維度：
  1. **語義相似度**：jieba 分詞 + Jaccard 相似度（30%）
  2. **地理距離**：PostGIS ST_DWithin 查詢半徑內事件（30%）
  3. **時間接近度**：小時差異（20%）
  4. **類型匹配**：災情種類（20%）
- 去重半徑依災害類型不同（50m ~ 200m）
- 相似度 0.50~0.80 之間呼叫 Claude 進行語義確認
- 同一事件 → 合併（描述合併、傷亡數字由 LLM 重新萃取）
- 不同事件 → 建立新的災情事件

### F11: 多層級地理編碼
- 多來源 Geocoding：Google Places → Nominatim → Google Geocoding API
- 地標模式偵測（「A 附近的 B」）
- 地點歧義消解（多候選地點）
- 精確度判斷與追問機制（最多 3 次）
- 進程級快取（最多 500 項）

---

## 非功能性需求
- 前端響應式設計（手機、桌面都能使用）
- Chat API 支援 streaming 回應（SSE 逐字顯示 AI 回覆）
- 管理端與民眾端獨立部署，各自獨立的前端應用
- 管理端點（PUT/DELETE/PATCH events、LLM logs）需 JWT 認證
- 民眾端 API（chat、GET events）免認證
